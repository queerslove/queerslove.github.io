由于最开始只是想要做一些小的实验，因此并没有将版本控制摆上台面；今天想到Git管理，却因为不明原因将整个项目删除。挽救工作不但没有成功却将残余的代码也一并清扫而空。（至今未明原因。）算是蛮惨痛的教训。

记录一下Unity的Git管理配置。

1. 新建项目，Edit - Project Settings - Editor - Version Control改为Visible Meta Files。
2. 删除Library与Temp文件夹。
3. 新建.gitignore文件如下。
4. 上传项目。

```
# =============== #
# Unity generated #
# =============== #
Temp/
Obj/
UnityGenerated/
Library/
Assets/AssetStoreTools*

# ===================================== #
# Visual Studio / MonoDevelop generated #
# ===================================== #
ExportedObj/
*.svd
*.userprefs
*.csproj
*.pidb
*.suo
*.sln
*.user
*.unityproj
*.booproj

# ============ #
# OS generated #
# ============ #
.DS_Store
.DS_Store?
._*
.Spotlight-V100
.Trashes
Icon?
ehthumbs.db
Thumbs.db
```
